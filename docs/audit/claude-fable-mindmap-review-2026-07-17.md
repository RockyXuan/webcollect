## 3. 整改与打磨项（按优先级，给 Codex 执行）

> 都不阻塞 V1.2.0 使用。P2 建议进下一个小版本（V1.2.1），P3 顺手做。

### P2-1 缺少"重置布局"入口（本轮唯一实质 UX 缺口）

**问题**：用户拖动节点后，偏移按布局×分项永久保留；拖乱之后没有任何恢复自动布局的办法（「适应画布」只调相机不清偏移），只能手动一个个拖回去。

**改法**：
1. `mindmap-view.tsx` 布局轨底部（四个布局按钮之后，加一条分隔线）增加「重置布局」按钮，样式沿用 `wc-mindmap-layout-button` 家族，图标用 lucide `RotateCcw`。
2. 点击行为：`setOffsets((cur) => ({ ...cur, [layout]: {} }))` → `markDirty()` → `fit(true)`。只清**当前布局×当前分项**的偏移，不动其他布局/分项，不动折叠状态。
3. 偏移为空时按钮置灰（`disabled`），`aria-label="重置当前布局的手动摆放"`。
4. 危险性低（视图状态可再拖回来），不需要确认弹窗；但要在按钮 title 里写明"仅重置当前布局的手动位置"。

**验收**：拖动 2 个节点 → 点重置 → 节点回到自动布局位且相机 refit；IDB 中该布局 offsets 为空、其他布局 offsets 保留；Vitest 给 `mindmap-view-state` 补一个"清空单布局偏移"的用例；e2e 补一条。

### P2-2 模式选择不跨会话记忆（需用户拍板的产品决策）

**现状**：`collectionViewMode` 是纯 React state，每次打开页面/新标签页都回经典模式。AGD 记录"经典模式仍默认启动"是有意设计——但"我上次明明在用导图"的用户会觉得没记住。

**建议**：仿照壁纸 `defaultMode` 的做法，把上次模式镜像到 localStorage（key 如 `webcollect_collection_view_mode`），启动时读取；首次无记录仍默认经典。**先问用户要不要，要了再做**：改动点只有 `page.tsx` / `newtab-app.tsx` 的 useState 初始化 + `handleCollectionViewModeChange` 写入，约 15 行。

**验收**：切到导图 → 刷新 → 仍在导图；清 localStorage → 回经典。

### P3-1 favicon 失败时图标区短暂空白

**问题**：`ReadOnlyCardIcon` 按候选列表逐个 `onError` 前进，网络差/离线时每个候选都要等失败，期间图标区空白，最后才落到字母兜底。

**改法**：字母兜底作为底层始终渲染（当前 `is-fallback` 的样式），`<img>` 绝对定位叠在上面、`onLoad` 前 `opacity: 0`，加载成功才盖住字母。纯 CSS/JSX 改动，`node-hover-preview.tsx` 的 `PreviewIcon` 同样处理。

**验收**：断网打开导图，所有卡片立即显示字母图标，无空白框。

### P3-2 双侧脑图 chip 侧向以根节点为参照

**问题**：`mindmap-view.tsx:715` 判断 chip 放左还是右，是拿节点中心与**根节点**中心比。深层左侧节点被用户拖到根右侧后，chip 会跳到另一边，与其连线方向不一致（纯观感问题）。

**改法**：侧向应由**顶级分类的分侧结果**决定：`layoutPositions` 的 bilateral 分支里已知每个顶级分类在 left/right 数组中的归属，把它记进 `MindmapLayoutResult`（如 `sideByNodeId: Record<string, "left"|"right">`，右侧布局全部 "right"），组件直接读，不再做几何比较。

**验收**：双侧布局下把左侧分类拖到根右边，chip 仍保持在左侧（跟随连线端），单测补一条 sideByNodeId 断言。

### P3-3 顺手项

- `mindmap-view.tsx` 里 `descendantCount(node)` 在渲染循环中对每个节点重算（O(n·深度)）；可在 `indexNodes` 一次性算好 Map。当前规模无感知，300+ 节点时省一点。
- 添加对话框以 Escape 关闭后，焦点应回到触发的「+」按钮（键盘可达性闭环）。

## 4. 明确不要做的

- 不要给拖拽加"吸附回原位/网格"——用户要的就是自由摆放。
- 不要把 `mindmapViewState` 接入云同步（视图状态本地化是既定架构，V2 再议 layout+collapsed 的轻量同步）。
- 不要因 P3-2 重构布局引擎——只是把已有分侧信息带出来。

## 5. 给 Codex 的执行顺序

P2-1（重置布局，半天）→ P3-1（favicon 兜底，1 小时）→ P3-2 + P3-3（半天）→ P2-2 等用户拍板后再做 → 合并入下一个功能小版本，照常走 CI/发版流程。每步照旧：一步一 commit、ts-check/lint/vitest/build:ext 全绿再进下一步。
